[SOLVED] Need help drawing a rectangle with opacity.
« on: May 16, 2016, 11:05:25 pm »
I need to draw a rectangle that can have a opacity value from 0 to 100.

The below snippet from Delphi Dabbler (http://delphidabbler.com/tips/203) works but is very slow with larger rectangles:

Code: Pascal  [Select][+][-]
  1. procedure DrawTransparentRectangle(Canvas: TCanvas; Rect: TRect;
  2.   Color: TColor; Transparency: Integer);
  3. var
  4.   X: Integer;
  5.   Y: Integer;
  6.   C: TColor;
  7.   R, G, B: Integer;
  8.   RR, RG, RB: Integer;
  9. begin
  10.   RR := GetRValue(Color);
  11.   RG := GetGValue(Color);
  12.   RB := GetBValue(Color);
  13.  
  14.   for Y := Rect.Top to Rect.Bottom - 1 do
  15.     for X := Rect.Left to Rect.Right - 1 do
  16.     begin
  17.       C := Canvas.Pixels[X, Y];
  18.       R := Round(
  19.         0.01 * (Transparency * GetRValue(C) + (100 - Transparency) * RR)
  20.       );
  21.  
  22.       G := Round(
  23.         0.01 * (Transparency * GetGValue(C) + (100 - Transparency) * RG)
  24.       );
  25.  
  26.       B := Round(
  27.         0.01 * (Transparency * GetBValue(C) + (100 - Transparency) * RB)
  28.       );
  29.  
  30.       Canvas.Pixels[X, Y] := RGB(R, G, B);
  31.     end;
  32. end;

I know the reason is because accessing pixels individually is a lot slower then using scanline but I am unsure how I exactly I could optimise the above. I thought maybe draw to a offscreen bitmap and use it's scanline but not exactly sure how I would go about that (the scanline part), I also need to make sure I can see through the opacity rectangle at whatever graphic is underneath on the canvas.

Alternatively, Delphi has Windows.AlphaBlend which I believe could help me greatly but I couldn't find this in Lazarus.

Can anyone advise how to write such a procedure to paint a rectangle with opacity that is not slow? I need to draw a overlay rectangle on top of a canvas, it doesn't even need to be based on the example above I just need a quick way of doing this without any 3rd party solution such as GR32 etc.

Thanks.

Try using BGRABitmap library.
Code: Pascal  [Select][+][-]
  1. procedure TForm1.BGRAVirtualScreen1Redraw(Sender: TObject; Bitmap: TBGRABitmap);
  2. begin
  3.   Bitmap.Rectangle(x, y, x2, y2, BGRA(0,0,0,alpha), dmDrawWithTransparency);
  4. end;

x, y, x2 and y2: here put your rect. BGRA function returns a color with alpha value. In this demo I'm also using BGRAControls BGRAVirtualScreen component that is a custom panel to draw into it.

Or you can use just a TBGRABitmap object and then draw to your canvas instead.
